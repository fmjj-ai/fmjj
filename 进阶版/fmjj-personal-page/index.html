    资料核验与维护建议

    本项目采用的原则

    • 优先官方空间、官方/资讯站资料帖、主流音乐平台、票务平台。
    • 不把高仿账号、无法验证的个人隐私、未经授权音源写入页面。
    • 平台粉丝数、活动排期和热门歌曲会变化，建议发布前重新核对。

    维护入口

    页面数据集中在 assets/js/data.js，原始结构化数据在 assets/data/。修改 JSON 后可重新生成 data.js，或直接编辑 data.js。

    部署到 GitHub Pages、Netlify、Cloudflare Pages 时，直接上传整个文件夹即可。
