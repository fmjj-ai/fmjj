# 封茗囧菌个人网页文件夹

打开方式：直接双击 `index.html`，或把整个文件夹放到任意静态服务器中。

## 文件结构

```text
fengmingjiongjun_site/
├─ index.html              # 主网页
├─ styles.css              # 页面样式，响应式布局与明暗主题
├─ script.js               # 渲染歌曲、平台、时间线和主题切换
├─ data.js                 # 前端直接读取的数据
├─ data/facts.json         # 结构化资料与来源
├─ assets/images/          # 本地占位图；真实照片/封面采用外链展示
├─ assets/icons/           # favicon
└─ prompts/                # 继续扩展网页/图片的提示词
```

## 设计定位

- 风格：粉白、淡黄、浅蓝的柔和音乐人主页，带轻微玻璃拟态。
- 布局：首页 Hero → 关键信息 → 个人资料 → 热门曲目 → 平台主页 → 照片/封面视觉 → 时间线 → 来源说明。
- 交互：明暗主题切换、歌曲搜索、平台卡片跳转。
- 响应式：桌面端、平板、手机均可浏览。

## 核验原则

1. 优先采用音乐平台艺人页、B站个人空间、抖音主页、Apple Music、Amazon Music、Spotify 等平台资料。
2. 萌娘百科资料较详细，但页面本身提示“资料需要长期更新”，因此只采用可公开展示且风险较低的项目。
3. 身高、体重、民族、居住地等资料没有放进首页，以避免过度个人化或误传。
4. 图片采用外链显示，不在文件夹中二次分发；网络不可用时会显示本地占位 SVG。

## 主要来源

- Apple Music 艺人页：生日、类型、最新发行、Top Songs。
- Amazon Music 艺人页：热门曲目、近期发行、封面图外链。
- QQ音乐歌手页：歌手主页、歌曲/专辑数量和热门歌曲线索。
- Bilibili个人空间：B站主页入口。
- 抖音主页：抖音号 mandisasa、音乐人身份、合作邮箱线索。
- 新浪平台信息：全网 ID、平台列表、代表作、合作联系方式。
- 萌娘百科：履历、代表作、粉丝称呼、时间线；已按其提示谨慎使用。
